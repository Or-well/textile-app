# Changelog

本文档记录 Textile 的用户可见变化、兼容性变化和重要修复。

格式参考 Keep a Changelog，版本号以根目录 `package.json` 为准。

## [0.9.1] - 2026-06-25

### Changed

- 默认角色权限、角色标签和权限分组改由 `src/config/permissions.json` 维护，启动时会校验未知权限、未知角色和 owner 锁定权限。

## [0.9.0] - 2026-06-25

### Added

- 普通修改包现在可以携带术语编辑和术语删除记录；导入时会校验导出者术语权限，并在主项目术语已变化时进入冲突处理。
- 普通成员默认可导入项目维护修改包，仍不默认获得导出维护修改或危险导入权限。

### Changed

- 非强制签名项目中，项目更新包可像普通修改包一样选择是否签名；界面不再把普通项目更新固定称为“带签名的项目更新包”。
- 非强制签名项目中，负责人转让不再强制要求当前负责人私钥、新负责人公钥或交接证明；转让会导出未签名项目更新包并提示接收方人工确认来源。
- 成员与角色权限管理的说明文案更明确，区分角色默认权限、成员额外允许和成员额外禁止。

## [0.8.6] - 2026-06-25

### Changed

- 单条词条编辑页点击“校对通过”后会像“保存并下一条”一样自动跳到下一条匹配词条。
- 默认“翻译”“校对”和“审核”角色的争议、批注和上下文处理权限已对齐；三者的翻译、校对、审核流程权限仍保持分离。

## [0.8.5] - 2026-06-24

### Added

- 强制签名项目导出 `.hproj` 备份时，如果当前发布负责人尚未登记有效身份公钥，会先提醒分发项目包前建议创建负责人身份密钥；负责人仍可选择继续导出纯备份。

## [0.8.4] - 2026-06-24

### Fixed

- 修复词条历史、文件历史、批注、术语创建人和修改包预览等位置直接显示底层成员 ID 的问题，统一显示成员名、禁用状态或“已删除成员”。

## [0.8.3] - 2026-06-24

### Added

- 普通修改包和任务范围修改包导出时可选择是否包含当前用户自己的登录密码修改；默认不包含，勾选后只传递密码哈希、盐和更新时间，导入端只合并导出人自己的凭据字段并继续阻止角色、权限、公钥等成员信息夹带。

## [0.8.2] - 2026-06-23

### Fixed

- 修复词条管理页在显示提示消息时批量操作栏和表格区域被压缩裁切的问题，并统一复用批量选择栏样式。

## [0.8.1] - 2026-06-23

### Fixed

- 修正文件导入弹窗的 JSON、CSV 和词条交换示例，使示例字段覆盖说话人并与当前导入规则一致。

## [0.8.0] - 2026-06-23

### Added

- 术语页新增多选批量管理，可以统一设置词性、大小写规则或删除术语，并在执行前显示可处理数量和跳过原因。
- 文件页新增既有文件多选批量管理，可以统一隐藏、锁定、调整分组或删除文件；批量删除会在一个写入计划中提交项目索引、源文件、词条目录和历史事件。
- 文件页新增分组筛选、按分组分段与折叠显示，并支持单文件调整分组、整组重命名和将全部文件移出分组；分组继续使用兼容旧项目的文件元数据，不创建磁盘目录。

## [0.7.1] - 2026-06-23

### Added

- Textile 会按项目和成员在本机记住最近编辑的文件及各文件最近查看的词条；程序重启后再次打开项目并登录时会恢复到该位置。

### Fixed

- 修复文件词条页点击左侧词条时列表反复跳动、分页范围随选中项偏移的问题；普通点击不再强制滚动，首次恢复和外部定位仍会把目标词条尽量放在第二行。

## [0.7.0] - 2026-06-23

### Added

- 文件页“更多 > 查看历史”现在会打开只读文件历史弹窗，汇总源文件、译文导入、文件状态、词条和批注相关事件。
- 文件词条页左侧列表新增组件内分页滚动，支持每页 20、50、100、200、500 或 800 条，并会在当前会话中恢复同一文件上次查看的词条。

### Fixed

- 词条编辑页改为复用公共工作台页头，保持标题、说明和右上角计数与其他工作台页面一致。
- 调整词条编辑页高度布局，避免页面根容器强制撑满视口导致与其他工作台页面的顶部节奏不一致。
- 文件词条页左侧列表改为更紧凑的分页行，中间编辑区压缩默认高度，减少常规词条编辑时的内部滚动。
- 修复文件词条页左侧列表没有形成明确内部滚动区、恢复目标词条时不能稳定停在第二行的问题。
- 词条页没有错误或提示时不再保留空消息行，筛选栏会紧跟公共页头，顶部高度与其他工作台页面保持一致。
- 文件页返回时会把最近查看文件尽量定位到列表第三行，前方文件不足两个时自动靠前显示。

## [0.6.2] - 2026-06-22

### Added

- 创建和编辑任务时支持把多个所选文件按文件顺序组成连续词条范围，并显示所选文件数和词条总数；多文件任务打开关联词条时可以选择具体文件。
- 词条右侧批注区和工作台批注页新增显示顺序选择，默认按由新到旧显示。
- 文件页会在返回时标注当前会话最近查看的文件，并自动把它定位到列表上方的第二行位置。

### Changed

- 任务进度改为按任务类型计算：翻译只判断有效译文，校对只判断指定轮次，审核只判断已审核；术语和自定义任务不再显示无法定义的进度。
- 任务类型文案统一使用“审核”，任务详情的统计改为只显示当前任务类型对应的已完成、未完成数量和争议数量。
- 词条右侧批注区移除重复的争议原因输入和“标记争议”按钮，回复输入框改为在被回复批注下方展开。
- 创建和编辑任务时不再显示提交方式选项，旧任务的提交方式字段继续保留用于兼容。
- 创建和编辑任务时不再显示单独的目标字段；任务详情只显示说明文本，旧任务目标会作为说明兜底展示。

### Fixed

- 修复任务编辑弹窗的多文件选择控件拉伸同一行输入框、导致类型和成员等控件高度不一致的问题。
- 修复词条批注面板重复显示“当前词条批注”标题和分割线导致信息层级冗余的问题。

## [0.6.1] - 2026-06-22

### Added

- 发布产物新增 `THIRD_PARTY_NOTICES.txt`，集中列出并附带 npm 与 Rust 第三方依赖的许可证文本；该文件随 Web 构建和 Tauri 安装包分发，但不增加程序内入口。

### Fixed

- 修复导出签名修改包前的密钥准备弹窗操作按钮横向排列时与下方说明文案不匹配的问题，改为纵向同宽按钮布局。

## [0.6.0] - 2026-06-22

### Added

- 身份密钥页面新增可信发布者恢复和“私钥丢失，重建信任”流程：存在其他可信发布者时可生成不写入项目的恢复密钥申请，由可信发布者签发公钥恢复项目更新包；所有可信发布私钥都不可用时，负责人可生成新私钥和新的 `.hproj`，提升项目信任代次并要求团队废弃旧副本。
- 签名密钥轮换、公钥恢复、密钥撤销和负责人转让的项目更新包会归档到项目的 `changes/transitions/`，可从身份密钥页面重新保存最近的过渡包。
- 负责人转让新增持钥证明文件；新负责人必须使用当前私钥生成绑定当前项目版本且 24 小时内有效的交接证明。

### Changed

- 导出需要签名的修改包时，如果当前成员尚未加载可用私钥，导出页会直接引导导入已有私钥文件或生成新的签名密钥，并在准备完成后继续导出；已有有效公钥的项目更新发布者只能导入与当前公钥匹配的旧私钥，不能在发布时直接换钥绕过签名链。
- 所有前端生成文件的保存流程改为必须走可确认保存结果的保存对话框；当前环境无法确认文件已保存时，不再使用浏览器原生下载兜底，也不会推进相关项目状态。
- 修改包、项目备份、成品包和签名信任过渡包会先确认保存目标，再执行打包生成；需要同时保存私钥和登记文件、过渡包或 `.hproj` 的流程会作为一组文件保存，任一文件未确认保存时不会提交项目状态。
- 桌面版生成文件保存改为调用 Tauri 原生保存对话框并由后端写入文件，保存成功后会拿到明确的保存结果和实际路径。
- 桌面版内置用户手册改由 Tauri 官方 opener 能力交给系统默认 PDF 阅读器打开，不再手写调用各平台外部命令。
- 签名密钥轮换、负责人密钥撤销和负责人转让改为通过同一写入计划提交成员信息、审计事件、过渡包归档和项目 revision；项目状态只在所有外部文件确认保存后切换。
- 普通成员在本地副本中新建签名密钥后，不再立即继续导出签名修改包；Textile 会同时保存公钥登记文件，要求负责人登记并通过可信项目更新分发公钥后再提交。

### Fixed

- 修复签名私钥只按成员 ID 缓存导致的断链风险；现在本机已加载私钥必须同时匹配项目中当前登记的公钥编号和公钥内容，才会被视为可用于签名、导出或发布项目更新包。
- 生成自己的签名密钥和负责人轮换密钥时，会先确认私钥文件保存成功，再把新公钥写入项目或启用新公钥；发布项目更新包缺少旧私钥时不再允许直接生成新公钥替代签名。
- 转让项目负责人时，会在转让区域直接显示缺少新负责人公钥、当前负责人私钥或权限不足等阻止原因。
- 调整负责人转让区域的交接证明导入控件样式，避免文件导入按钮被拉伸成输入框，并补充当前交接证明状态提示。
- 有发布项目更新包权限的负责人撤销自己的身份密钥时，会先导出由旧密钥签名且标记公钥撤销的项目更新包，确认保存后才撤销本地主项目密钥。
- 强制签名项目中新增成员并生成一次性私钥时，会先保存私钥文件，确认保存成功后才把新成员和公钥写入项目，避免私钥文件丢失但项目已登记公钥。
- 阻止撤销项目中最后一把可信发布密钥；不同信任代次的项目副本不能通过普通项目更新包跨代接收，会明确要求重新获取可信 `.hproj`。

## [0.5.0] - 2026-06-22

### Added

- 成员管理支持重新启用已禁用成员，以及在输入成员名确认后永久删除已禁用成员账户；启用保留原账户配置，永久删除保留历史数据中的成员 ID 和删除审计事件。
- 词条编辑页支持按独立权限锁定、解锁、隐藏和取消隐藏词条，并记录管理状态事件。
- 身份密钥页面支持主动卸载当前运行内存中的私钥。
- 非强制签名项目导出普通修改包时，可以选择是否给本次修改包签名。
- 有发布项目更新包权限的负责人轮换自己的身份密钥时，会先导出由旧密钥签名且包含新公钥的项目更新包，确认保存后才切换本地主项目密钥。
- 转让项目负责人时，会先导出由旧负责人密钥签名且包含新负责人公钥的项目更新包，确认保存后才完成本地主项目转让。

### Changed

- 界面和文档中的协作讨论文案统一改为“批注”，保留内部 `Comment` 类型、`comments/` 存储路径和路由兼容。
- 普通修改包现在会随词条导入批注状态更新、批注删除事件、词条上下文和争议信息；预览与冲突处理会分别显示批注、上下文和争议信息变化。
- 设置页侧边栏“我的身份密钥”改为“身份密钥”，页面标题仍保留“我的身份密钥”用于强调当前成员视角。
- 任务页面和任务 service 共用统一状态转换规则；成员筛选和负责人显示会保留已禁用成员及已删除成员 ID。
- 旧项目的角色权限首次按新版权限规则读取时会兼容补充词条锁定和隐藏权限；保存角色权限后按当前配置精确生效。

### Fixed

- 修复切换词条后可能绕过所属文件锁定或隐藏状态继续写入的问题；保存、译文导入、历史恢复、上下文和争议操作现在都会在 service 层重新检查。
- 修复任务操作按钮可能在不合法状态下显示，点击后才由 service 拒绝的问题。

### Security

- 已撤销的身份密钥不能通过重新导入同一私钥或公钥恢复；退出登录、切换项目和移除项目会清除相应的内存私钥。
- 普通成员现在可以撤销自己的身份密钥；撤销其他成员身份密钥仍需要密钥管理权限。
- 项目更新包仍只使用接收端当前项目已信任的公钥验签，不会使用包内新公钥验证自身。

## [0.4.1] - 2026-06-21

### Changed

- 词条管理页的批量操作下拉改为按目标状态显示“已审核”“已校对”“已翻译”“有争议”“无争议”，不再暴露校对通过、退回和解决争议等流程动作文案。
- 空白原文词条允许使用空白译文进入已翻译、已校对和已审核流程；普通原文仍要求译文包含可见内容。
- 项目工作台各主页面统一使用公共页头样式，并补充页面说明文案。

### Fixed

- 修复词条管理页批量操作完成提示出现后，批量操作栏被表格滚动区挤压或覆盖的问题。

## [0.4.0] - 2026-06-21

### Added

- 新增全宽词条管理页，支持跨文件搜索、组合筛选、排序、分页、跨页选择，以及批量校对、审核、回退和争议处理。
- 批量操作会在执行前显示可处理数量和跳过原因，并在执行时重新校验权限、任务范围和工作流状态；相关词条、批注和历史事件通过同一写入计划提交。
- 文件页可以导出可往返导入的 Textile 词条交换 JSON/JSONL，保留词条内容和翻译、校对、审核状态，但不携带锁定、隐藏、分配、争议、批注或权限数据。

### Changed

- 源 JSON/JSONL 在提供工作流字段时会严格校验状态、译文和校对次数；旧 JSON/JSONL、CSV、TXT、KS 继续按原内容格式兼容导入。
- 导入帮助和成品导出增加用途提示，明确普通译文导入只更新译文，成品文件不用于恢复 Textile 工作流。

## [0.3.2] - 2026-06-21

### Changed

- 禁止自校对改为比较当前有效译文流程的首位翻译者；校对或审核时修改译文不再覆盖 `translated_by`。
- 审核限制语义改为禁止最近一轮校对的首位校对者审核自己校对的译文，并同步设置项文案。
- 项目许可证明确为 Apache-2.0。
- 工作台侧栏“帮助”改名为“使用手册”，设置页关于区域增加许可证入口和 Apache-2.0 全文查看弹窗。
- 签名密钥页面明确区分公钥和私钥，支持导出/导入公钥登记文件；强制签名项目新增成员时可同步生成一次性私钥文件。
- 成品导出、术语导出和导入示例文件下载统一使用保存对话框或明确下载确认提示，避免误导文件已保存位置。

## [0.3.1] - 2026-06-21

### Changed

- 创建项目时可以设置译者自校、自审和同一成员多轮校对开关，新项目创建页默认勾选允许。

## [0.3.0] - 2026-06-21

### Changed

- 任务功能关闭时，任务导航入口会隐藏，任务 service 也会拒绝新增、更新、删除、分配、提交、完成、收回和重开操作。
- 任务状态变更改为集中状态转换表控制，任务进度支持按 `entry_ids` 或多个文件关联；新建和编辑任务时可选择多个文件。
- CSV 导入改用统一解析器，支持引号、逗号、转义引号和多行字段；源文件导入会阻止重复 key。
- 成品导出、术语导出和 `.hproj` 备份增加 service 层权限兜底。
- LocalStorage 读写集中使用容错包装；项目配置读取会拒绝不支持的未来 schema，并规范化旧 v1 配置。
- 新建或编辑任务时，截止时间会按所选 IANA 时区转换为 UTC 保存，并记录任务原始时区；跨时区成员查看时显示本地时间和任务时区时间。
- 批注、历史、术语、文件、密钥、修改包和程序更新等时间统一按当前设备时区格式化，时间排序统一按绝对时刻处理。
- 禁止自校对时只比较当前有效译文的最新译者；禁止自审核时只比较最新一条校对记录，不再因成员曾翻译或参与过更早轮次校对而阻止操作。
- 新建项目默认要求修改包带有有效成员签名；有项目管理权限的成员可在“协作与备份”中调整，旧项目缺少该设置时继续允许未签名普通包。

- 程序更新只在实际写入、导入导出或存在未保存草稿时暂缓，不再根据当前所在页面推断正在编辑。
- 桌面版更新统一使用“安装并重启”文案；PWA 更新由用户明确点击“刷新并应用”，不再在后台自动刷新。
- 修改包和 `.hproj` 备份只在用户确认文件已保存后提交导出状态；支持系统保存对话框时以文件写入完成为准。
- 删除带回复的批注前会显示级联删除数量；普通成员不能通过删除自己的父批注连带删除其他成员的回复。
- 译文与工作流变更统一记录操作类型和完整前后快照：普通翻译编辑会重置后续流程，校对或审核中修改译文则保留并推进当前流程；批量导入和普通修改包合并也会生成本地版本记录。
- 统计进度在关闭审核或校对次数为 0 时会忽略对应阶段权重，并按剩余阶段重新归一化。
- 普通成员导出普通修改包时默认限制在自己已分配任务范围内，并携带自己修改的词条、争议状态、批注和任务执行状态；负责人、管理员或有任务管理权限的成员可以导出其他成员任务修改，导入端只对有效签名的管理者普通包放宽任务范围。
- 普通修改包导入只接受词条译文、工作流字段和争议字段，任务修改只接受已有任务的执行状态；锁定、隐藏、分配、任务范围等管理字段变化会被阻止。
- 项目更新包导入改为通过同一写入计划提交内容、日志和陈旧文件清理，并继续把 `project.json` 放在最后写入。

### Fixed

- 修复任务可从不合法状态直接完成、重开或提交，以及任务范围可能引用不存在词条的问题。
- 修复任务截止时间直接保存 `datetime-local` 导致跨时区含义变化，以及夏令时缺失或重复时刻未校验的问题；旧无时区截止时间会保留并要求编辑者确认来源时区。
- 修复“记住登录”期限受夏令时切换影响，以及无效过期时间可能被当作未过期的问题。
- 修复术语目标匹配未遵守 `case_sensitive` 的问题。
- 修复“禁止审核自己的译文”错误按译者判断的问题；审核现在只禁止最新校对者审核自己的校对结果。

- 修复未进行编辑时仍提示需要完成当前编辑，以及更新状态说明重复显示的问题。
- 修复导入同名源文件时共用并覆盖 `source_path`，以及删除其中一个文件可能误删共享源文件的问题。
- 修复任务 JSONL 损坏时被静默当成空列表并可能被后续保存覆盖的问题。
- 修复已审核词条可被普通保存直接修改译文，以及普通保存可能顺带覆盖锁定、隐藏、分配等管理字段的问题。
- 修复导入译文会覆盖锁定或隐藏词条，以及清空译文后统计和流程状态仍可能按后续阶段计算的问题。
- 修复普通修改包可能覆盖词条和任务管理字段，以及普通包冲突检测没有覆盖校对/审核审计字段的问题。
- 修复项目更新包导入过程中部分文件已写入但项目索引尚未推进时缺少统一回滚计划的问题。
