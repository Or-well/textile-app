# Changelog

本文档记录 Textile 的用户可见变化、兼容性变化和重要修复。

格式参考 Keep a Changelog，版本号以根目录 `package.json` 为准。

## [Unreleased]

### Changed

- 程序更新只在实际写入、导入导出或存在未保存草稿时暂缓，不再根据当前所在页面推断正在编辑。
- 桌面版更新统一使用“安装并重启”文案；PWA 更新由用户明确点击“刷新并应用”，不再在后台自动刷新。
- 修改包和 `.hproj` 备份只在用户确认文件已保存后提交导出状态；支持系统保存对话框时以文件写入完成为准。
- 删除带回复的评论前会显示级联删除数量；普通成员不能通过删除自己的父评论连带删除其他成员的回复。
- 译文被手工保存、批量导入或通过普通修改包合并后，只要内容发生变化，就会统一回到“已翻译”或“未翻译”，并清空后续校对和审核记录。
- 统计进度在关闭审核或校对次数为 0 时会忽略对应阶段权重，并按剩余阶段重新归一化。

### Fixed

- 修复未进行编辑时仍提示需要完成当前编辑，以及更新状态说明重复显示的问题。
- 修复导入同名源文件时共用并覆盖 `source_path`，以及删除其中一个文件可能误删共享源文件的问题。
- 修复任务 JSONL 损坏时被静默当成空列表并可能被后续保存覆盖的问题。
- 修复已审核词条可被普通保存直接修改译文，以及普通保存可能顺带覆盖锁定、隐藏、分配等管理字段的问题。
- 修复导入译文会覆盖锁定或隐藏词条，以及清空译文后统计和流程状态仍可能按后续阶段计算的问题。

## [0.2.3] - 2026-06-20

### Fixed

- 修复发布版无法打开内置 PDF 用户手册的问题。

## [0.2.2] - 2026-06-20

### Added

- 工作台侧边栏增加“帮助”入口，可打开随程序内置的 PDF 用户手册。

### Fixed

- 修复 Web/PWA 版本清单资源 hash 受换行符差异影响，导致 GitHub Actions 发布检查反复失败的问题。
- 修复桌面版已是最新版本时更新结果仍显示最新版本和构建编号尚未获取的问题。

## [0.2.1] - 2026-06-20

### Changed

- 启动页当前项目摘要增加校对进度。

### Fixed

- 修复最近项目在浏览器需要重新确认文件夹权限时直接要求重新选择目录的问题。
- 修复词条编辑页搜索或筛选结果较多时左侧列表无法正常滚动的问题，并同步约束编辑器和右侧辅助面板的桌面端滚动区域。

## [0.2.0] - 2026-06-20

### Changed

- 项目概览中的修改包快捷入口统一使用“修改包”术语。
- “导出普通修改包”和“导入修改包”会直接定位到导入导出页的对应区域。
- 移除尚无持久化队列支持的“查看待合并修改”入口。

### Fixed

- 修复修改包快捷入口定位时目标标题被工作台粘性顶栏遮挡的问题。
