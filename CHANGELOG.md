# Changelog

本文档记录 Textile 的用户可见变化、兼容性变化和重要修复。

格式参考 Keep a Changelog，版本号以根目录 `package.json` 为准。

## [Unreleased]

### Added

- 成员管理支持重新启用已禁用成员，以及在输入成员名确认后永久删除已禁用成员账户；启用保留原账户配置，永久删除保留历史数据中的成员 ID 和删除审计事件。
- 词条编辑页支持按独立权限锁定、解锁、隐藏和取消隐藏词条，并记录管理状态事件。
- 身份密钥页面支持主动卸载当前运行内存中的私钥。
- 非强制签名项目导出普通修改包时，可以选择是否给本次修改包签名。

### Changed

- 界面和文档中的协作讨论文案统一改为“批注”，保留内部 `Comment` 类型、`comments/` 存储路径和路由兼容。
- 普通修改包现在会随词条导入批注状态更新、批注删除事件、词条上下文和争议信息；预览与冲突处理会分别显示批注、上下文和争议信息变化。
- 设置页侧边栏“我的身份密钥”改为“身份密钥”，页面标题仍保留“我的身份密钥”用于强调当前成员视角。
- 任务页面和任务 service 共用统一状态转换规则；成员筛选和负责人显示会保留已禁用成员及已删除成员 ID。
- 旧项目的角色权限首次按新版权限规则读取时会兼容补充词条锁定和隐藏权限；保存角色权限后按当前配置精确生效。

### Fixed

- 修复切换词条后可能绕过所属文件锁定或隐藏状态继续写入的问题；保存、译文导入、历史恢复、上下文和争议操作现在都会在 service 层重新检查。
- 修复任务操作按钮可能在不合法状态下显示，点击后才由 service 拒绝的问题。

### Security

- 已撤销的身份密钥不能通过重新导入同一私钥或公钥恢复；退出登录、切换项目和移除项目会清除相应的内存私钥。
- 普通成员现在可以撤销自己的身份密钥；撤销其他成员身份密钥仍需要密钥管理权限。

## [0.4.1] - 2026-06-21

### Changed

- 词条管理页的批量操作下拉改为按目标状态显示“已审核”“已校对”“已翻译”“有争议”“无争议”，不再暴露校对通过、退回和解决争议等流程动作文案。
- 空白原文词条允许使用空白译文进入已翻译、已校对和已审核流程；普通原文仍要求译文包含可见内容。
- 项目工作台各主页面统一使用公共页头样式，并补充页面说明文案。

### Fixed

- 修复词条管理页批量操作完成提示出现后，批量操作栏被表格滚动区挤压或覆盖的问题。

## [0.4.0] - 2026-06-21

### Added

- 新增全宽词条管理页，支持跨文件搜索、组合筛选、排序、分页、跨页选择，以及批量校对、审核、回退和争议处理。
- 批量操作会在执行前显示可处理数量和跳过原因，并在执行时重新校验权限、任务范围和工作流状态；相关词条、批注和历史事件通过同一写入计划提交。
- 文件页可以导出可往返导入的 Textile 词条交换 JSON/JSONL，保留词条内容和翻译、校对、审核状态，但不携带锁定、隐藏、分配、争议、批注或权限数据。

### Changed

- 源 JSON/JSONL 在提供工作流字段时会严格校验状态、译文和校对次数；旧 JSON/JSONL、CSV、TXT、KS 继续按原内容格式兼容导入。
- 导入帮助和成品导出增加用途提示，明确普通译文导入只更新译文，成品文件不用于恢复 Textile 工作流。

## [0.3.2] - 2026-06-21

### Changed

- 禁止自校对改为比较当前有效译文流程的首位翻译者；校对或审核时修改译文不再覆盖 `translated_by`。
- 审核限制语义改为禁止最近一轮校对的首位校对者审核自己校对的译文，并同步设置项文案。
- 项目许可证明确为 Apache-2.0。
- 工作台侧栏“帮助”改名为“使用手册”，设置页关于区域增加许可证入口和 Apache-2.0 全文查看弹窗。
- 签名密钥页面明确区分公钥和私钥，支持导出/导入公钥登记文件；强制签名项目新增成员时可同步生成一次性私钥文件。
- 成品导出、术语导出和导入示例文件下载统一使用保存对话框或明确下载确认提示，避免误导文件已保存位置。

## [0.3.1] - 2026-06-21

### Changed

- 创建项目时可以设置译者自校、自审和同一成员多轮校对开关，新项目创建页默认勾选允许。

## [0.3.0] - 2026-06-21

### Changed

- 任务功能关闭时，任务导航入口会隐藏，任务 service 也会拒绝新增、更新、删除、分配、提交、完成、收回和重开操作。
- 任务状态变更改为集中状态转换表控制，任务进度支持按 `entry_ids` 或多个文件关联；新建和编辑任务时可选择多个文件。
- CSV 导入改用统一解析器，支持引号、逗号、转义引号和多行字段；源文件导入会阻止重复 key。
- 成品导出、术语导出和 `.hproj` 备份增加 service 层权限兜底。
- LocalStorage 读写集中使用容错包装；项目配置读取会拒绝不支持的未来 schema，并规范化旧 v1 配置。
- 新建或编辑任务时，截止时间会按所选 IANA 时区转换为 UTC 保存，并记录任务原始时区；跨时区成员查看时显示本地时间和任务时区时间。
- 批注、历史、术语、文件、密钥、修改包和程序更新等时间统一按当前设备时区格式化，时间排序统一按绝对时刻处理。
- 禁止自校对时只比较当前有效译文的最新译者；禁止自审核时只比较最新一条校对记录，不再因成员曾翻译或参与过更早轮次校对而阻止操作。
- 新建项目默认要求修改包带有有效成员签名；有项目管理权限的成员可在“协作与备份”中调整，旧项目缺少该设置时继续允许未签名普通包。

- 程序更新只在实际写入、导入导出或存在未保存草稿时暂缓，不再根据当前所在页面推断正在编辑。
- 桌面版更新统一使用“安装并重启”文案；PWA 更新由用户明确点击“刷新并应用”，不再在后台自动刷新。
- 修改包和 `.hproj` 备份只在用户确认文件已保存后提交导出状态；支持系统保存对话框时以文件写入完成为准。
- 删除带回复的批注前会显示级联删除数量；普通成员不能通过删除自己的父批注连带删除其他成员的回复。
- 译文与工作流变更统一记录操作类型和完整前后快照：普通翻译编辑会重置后续流程，校对或审核中修改译文则保留并推进当前流程；批量导入和普通修改包合并也会生成本地版本记录。
- 统计进度在关闭审核或校对次数为 0 时会忽略对应阶段权重，并按剩余阶段重新归一化。
- 普通成员导出普通修改包时默认限制在自己已分配任务范围内，并携带自己修改的词条、争议状态、批注和任务执行状态；负责人、管理员或有任务管理权限的成员可以导出其他成员任务修改，导入端只对有效签名的管理者普通包放宽任务范围。
- 普通修改包导入只接受词条译文、工作流字段和争议字段，任务修改只接受已有任务的执行状态；锁定、隐藏、分配、任务范围等管理字段变化会被阻止。
- 项目更新包导入改为通过同一写入计划提交内容、日志和陈旧文件清理，并继续把 `project.json` 放在最后写入。

### Fixed

- 修复任务可从不合法状态直接完成、重开或提交，以及任务范围可能引用不存在词条的问题。
- 修复任务截止时间直接保存 `datetime-local` 导致跨时区含义变化，以及夏令时缺失或重复时刻未校验的问题；旧无时区截止时间会保留并要求编辑者确认来源时区。
- 修复“记住登录”期限受夏令时切换影响，以及无效过期时间可能被当作未过期的问题。
- 修复术语目标匹配未遵守 `case_sensitive` 的问题。
- 修复“禁止审核自己的译文”错误按译者判断的问题；审核现在只禁止最新校对者审核自己的校对结果。

- 修复未进行编辑时仍提示需要完成当前编辑，以及更新状态说明重复显示的问题。
- 修复导入同名源文件时共用并覆盖 `source_path`，以及删除其中一个文件可能误删共享源文件的问题。
- 修复任务 JSONL 损坏时被静默当成空列表并可能被后续保存覆盖的问题。
- 修复已审核词条可被普通保存直接修改译文，以及普通保存可能顺带覆盖锁定、隐藏、分配等管理字段的问题。
- 修复导入译文会覆盖锁定或隐藏词条，以及清空译文后统计和流程状态仍可能按后续阶段计算的问题。
- 修复普通修改包可能覆盖词条和任务管理字段，以及普通包冲突检测没有覆盖校对/审核审计字段的问题。
- 修复项目更新包导入过程中部分文件已写入但项目索引尚未推进时缺少统一回滚计划的问题。
